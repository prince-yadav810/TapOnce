## 1. Executive Summary

### 1.1 Product Overview
The NFC Smart Card Platform is a digital business card solution that bridges physical and digital networking. Customers receive a physical PVC NFC card (NTAG216 chip) that, when tapped with a smartphone, opens their personalized digital profile in a mobile browser. The platform enables customers to update their profile information in real-time without needing to reprint physical cards.

### 1.2 Business Model
- **Target Market:** India (initially Indore and Mumbai)
- **Customer Segment:** B2C - Individual professionals, entrepreneurs, corporate employees
- **Revenue Streams:**
  - Direct card sales via website
  - Agent-driven sales with commission structure
  - Potential future: Premium subscriptions for advanced features

### 1.3 Success Metrics
- Primary: Total cards sold per month
- Secondary: Agent recruitment rate, customer satisfaction, average order value
- Operational: Average fulfillment time (order to delivery)

---

## 2. Product Vision & Goals

### 2.1 Vision Statement
To revolutionize professional networking in India by making digital business cards accessible, affordable, and instantly updatable through NFC technology.

### 2.2 Product Goals
1. **Simplicity:** Enable non-technical users to manage their digital presence without training
2. **Speed:** Deliver cards within 7 days from order to customer delivery
3. **Scalability:** Support a distributed sales force through agent management system
4. **Reliability:** Ensure 99.9% uptime for tap view pages (critical for first impressions)

### 2.3 Out of Scope (For MVP)
- Mobile applications (iOS/Android native apps)
- Analytics/tap tracking for customers
- Multi-card management per customer
- International markets outside India
- Integration with third-party CRM systems

## 5. Technical Constraints

### 5.1 NFC Hardware Limitations

**NTAG216 Chip Specifications:**
- **Total Memory:** 888 bytes
- **Usable Memory (for URL):** ~868 bytes
- **Data Structure:** NDEF (NFC Data Exchange Format) message containing a URL record

**URL Length Constraints:**
- Maximum URL length: ~850 characters (theoretical)
- Practical limit: Keep URLs under 100 characters for reliability
- **Recommended URL Format:** `yourplatform.com/[username]` (30-50 chars typical)

**Implications for Platform:**
- URLs must be SHORT (cannot store full profile data on chip)
- URL is READ-ONLY after writing (cannot be updated remotely)
- One URL per card (cannot store multiple records)
- Must use web-based profile (chip only stores pointer to web page)

**URL Strategy:**
```
Format: https://yourplatform.com/[slug]

Examples:
- yourplatform.com/rahul-verma (personal name)
- yourplatform.com/tech-solutions (company name)
- yourplatform.com/c/ABC123 (fallback: Card ID)

Length: 30-60 characters typical
```

### 5.2 Mobile Network Constraints (India)

**Target Performance:**
- Page load on 3G: <3 seconds
- Page load on 4G: <1.5 seconds
- Page load on WiFi: <1 second

**Optimization Requirements:**
- Compress all images (WebP format, <100KB per image)
- Minify HTML/CSS/JS
- Use CDN for static assets
- Implement aggressive browser caching
- Lazy-load non-critical content

### 5.3 Browser Compatibility

**Must Support:**
- Chrome (Android) - 80%+ of taps expected
- Safari (iOS) - 15%+ of taps expected
- Samsung Internet (Samsung devices)
- Firefox, Edge (minimal but possible)

**Compatibility Requirements:**
- NFC tap must work on all Android 5.0+ devices
- iOS 13+ (iPhone XR and newer for background NFC reading)
- Fallback: QR code for devices without NFC

### 5.4 Device Constraints

**NFC Writing Hardware (Admin Side):**
- USB NFC writer device (e.g., ACR122U, PN532)
- Manual writing process: Admin copies URL â†’ writes to card using NFC writer software
- Batch writing not feasible (manual process per card)

**Customer Devices (Tap Side):**
- No app installation required (browser-based experience)
- Works on locked screen (Android) or requires app open (iOS 13-14) or background tap (iOS 14.5+)

   - Enters payment status (Paid/Advance/COD)

   - Wekonnect receives order

   - Wekonnect replies on WhatsApp: "Order confirmed, will ship by [date]"

## 9. Non-Functional Requirements

### 9.1 Performance
- **Page Load Time:** Tap view pages must load in <3 seconds on 3G networks
- **Database Query Time:** All dashboard queries must return in <500ms
- **Image Optimization:** All images compressed to <100KB
- **Concurrent Users:** System must support 100+ simultaneous dashboard users
- **Uptime:** 99.9% availability (critical for tap view pages)

### 9.2 Security
- **Password Storage:** Bcrypt hashing with salt (never store plain text)
- **HTTPS:** All pages served over SSL/TLS
- **SQL Injection Prevention:** Parameterized queries, input sanitization
- **XSS Prevention:** Escape all user-generated content
- **CSRF Protection:** Token-based form submission
- **Admin Authentication:** Secure session management, auto-logout after 30 minutes inactivity
- **File Upload Validation:** Image type verification, malware scanning (future)

### 9.3 Scalability
- **Database:** Design for 10,000+ customers, 100+ agents without performance degradation
- **Image Storage:** Use cloud storage (AWS S3, Cloudflare R2) for scalability
- **CDN:** Static assets served via CDN for global speed
- **Horizontal Scaling:** Architecture supports adding more servers as traffic grows

### 9.4 Reliability
- **Data Backup:** Daily automated database backups (retained for 30 days)
- **Error Logging:** Centralized error tracking (Sentry or similar)
- **Graceful Degradation:** If image fails to load, show placeholder (not broken layout)
- **Failover:** If primary database fails, switch to replica (future)

### 9.5 Usability
- **Mobile-First:** All dashboards optimized for smartphone use
- **Accessibility:** WCAG 2.1 AA compliance (alt text, keyboard navigation, screen reader support)
- **Browser Support:** Chrome, Safari, Firefox, Edge (latest 2 versions)
- **Offline Graceful Handling:** If user loses connection, show friendly message (not crash)

### 9.6 Compatibility
- **NFC Standards:** NDEF format, Type 2 Tag (NTAG216 compliant)
- **vCard Standard:** VCF 3.0 format for maximum device compatibility
- **PDF Standard:** PDF/A for archival compatibility
- **Image Formats:** WebP with JPG/PNG fallback for older browsers
